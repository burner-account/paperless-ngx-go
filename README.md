# paperless-ngx-go

Golang client library for the paperless-ngx API.

Based upon client generated by oapi-codegen from the paperless-ngx OpenAPI spec, API version 9.

## quickstart

```
client, err := paperless.NewXClientWithToken(
    "https://paperless-ngx.localdomain:8000",
    "api-token-generated-by-paperless-ngx",
)
// error handling

ctx, cancel := context.WithTimeout(context.Background(), 2 * time.Second)
defer cancel()

status, err := client.StatusRetrieveWithResponse(ctx)
// error handling

fmt.Println(status.JSON200.Database.Type)
fmt.Println(status.JSON200.Database.Status)
```

## examples

See `tests/` folder.

## testing

required:
- docker with compose plugin

The tests are run against testcontainers with a paperless-ngx-postgres-tika docker compose deployment.

To run the tests, execute
```
go test -v -timeout 5m -run ./...
```
or 
```
make test
```

On first run, the whole paperless-ngx docker deployment images will be downloaded. As soon as the API is healthy (another 30s), the tests inside the `tests/` folder are executed.

Testing currently covers far from all API calls. Feel free to add to it.


## re-generating the client

In order to generate the client from the paperless-ngx OpenAPI spec, run
```
make generate
```
This will
- remove old generated files
- create a patched OpenAPI spec (see patch/api.yaml.diff)
- generate client code via oapi-codegen (go tool)
- patch remaining container pointers in client (see patch/de-ptrize.diff)
- cleanup tmp files


## oapi-codegen client

Users familiar with clients generated by oapi-codegen can use the unmodified `Client` instead of the customized `XClient`.

```
    var opts []ClientOption = ...
    client, err := NewClient(
        "https://paperless-ngx.localdomain:8000",
        opts...,
    )

```
